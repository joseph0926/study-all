# study-all

체계적 학습 레포. 개발 기술과 영어를 매일 축적한다.

---

## 레포 구조

```
study-all/
├── CLAUDE.md                          # 이 파일 (게이트)
├── guides/
│   ├── dev-study.md                   # 개발 학습 가이드
│   └── english-study.md               # 영어 학습 가이드
├── .claude/commands/
│   ├── learn.md                       # /learn — 소스 기반 Q&A 튜터링
│   ├── study-skill.md                 # /study-skill — 레퍼런스 검증/개선
│   └── eg-d.md                        # /eg-d — 영어 데일리 루틴
├── docs/
│   ├── {skill-name}/                  # 개발 학습 기록
│   │   ├── plan.md
│   │   └── {Topic-Name}.md
│   └── english/                       # 영어 학습 기록
│       ├── expressions.md
│       └── daily/{YYYY-MM-DD}.md
└── ref/                               # 소스 코드/공식 문서
    ├── {name}-fork/
    └── {name}.dev/
```

## 도메인 라우팅

사용자 요청이 어떤 도메인에 해당하는지 판단하고, 해당 가이드를 읽어서 따른다.

| 신호 | 도메인 | 가이드 | 커맨드 |
|------|--------|--------|--------|
| 소스 코드, 기술 토픽, 스킬, 프레임워크, 라이브러리 | **개발** | `guides/dev-study.md` | `/learn`, `/study-skill` |
| 영어, English, 쉐도잉, 표현, 인터뷰 연습, streak | **영어** | `guides/english-study.md` | `/eg-d` |

- 커맨드(`/learn`, `/study-skill`, `/eg-d`)가 호출되면 해당 가이드를 자동으로 참조한다.
- 커맨드 없이 자유 대화로 진입해도, 위 신호를 기반으로 적절한 가이드를 읽고 따른다.
- 두 도메인에 걸치는 요청(예: "React 문서를 영어로 읽기 연습")은 양쪽 가이드를 모두 참조한다.

## 공통 원칙

- **사용자 주도**: AI는 튜터/코치. 일방적으로 작성하지 않는다.
- **세션 연속성**: 매 세션 시작 시 기존 `docs/` 기록을 확인하고 맥락을 복원한다.
- **기록 자동화**: 학습 기록은 AI가 정리하여 `docs/` 하위에 저장한다. 사용자에게 기록 부담을 주지 않는다.
- **최소 변경**: `docs/` 하위와 지정된 파일만 생성/수정한다. 다른 디렉토리는 건드리지 않는다.
